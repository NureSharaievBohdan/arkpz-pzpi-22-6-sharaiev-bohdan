1.1	Background (Передумови)
Радіація є серйозною загрозою для здоров’я людей та навколишнього середовища, особливо у зонах підвищеного ризику, таких як території атомних станцій, промислових об’єктів чи місця техногенних аварій. Метою розробки програмної системи є створення автоматизованого інструменту для збору, обробки та передачі даних про рівень радіації в реальному часі. Це дозволить користувачам, зокрема екологам і державним службам, отримувати актуальну інформацію, швидко реагувати на небезпеки та аналізувати історичні зміни для підвищення безпеки. Така система сприятиме зменшенню ризиків, своєчасному інформуванню населення і покращенню екологічного моніторингу.

1.2 Business Opportunities (Бізнес-можливості)
Існуючі аналоги, як-от стаціонарні монітори або мобільні додатки, мають недоліки: висока вартість, складність обслуговування, відсутність оперативного доступу. Наш продукт автоматизує збір даних через IoT-сенсори, забезпечує сповіщення в реальному часі, збереження історії вимірювань і доступ через мобільні та веб-застосунки.
Цільовий ринок — Україна, де моніторинг радіації є критично важливим через екологічні ризики. Користувачі: громадяни, державні органи, підприємства. Монетизація: передплата, платні ліцензії, продаж сенсорів, співпраця з компаніями.

2.1 Vision Statement
Програмна система для моніторингу радіації — це комплексне рішення, призначене для автоматичного збору, обробки та візуалізації даних про рівень радіації в реальному часі. Основна мета продукту — підвищення безпеки людей і довкілля шляхом оперативного інформування про небезпечні рівні радіації та надання інструментів для аналізу й моніторингу.
2.2 Major Features
-	MF-1: Відображення рівня радіації в реальному часі.
-	MF-2: Сповіщення про перевищення допустимих рівнів радіації.
-	MF-3: Історія вимірювань для аналізу змін радіаційного фону.
-	MF-4: Рекомендації для користувачів у разі підвищеної радіації.
-	MF-5: Створення звітів про радіаційний фон у вибраних зонах.
-	MF-6: Інтеграція з IoT-сенсорами для автоматичного збору даних.
-	MF-7: Веб-інтерфейс для перегляду даних та управління налаштуваннями.
-	MF-8: API для доступу до даних інших систем і додатків. 

3.1 Scope of Initial Release (Рамки первинного випуску)
Серверна частина:
-	Реалізація прийому даних про рівень радіації від IoT-сенсорів у реальному часі з перевіркою їх коректності, видаленням дублікатів і зберіганням у базі даних із резервним копіюванням.
-	Надання функціональності для управління користувачами, створення облікових записів, призначення ролей доступу, а також налаштування та моніторинг стану IoT-сенсорів.
-	Встановлення порогових значень для сповіщень про небезпечний рівень радіації.
-	Забезпечення безпеки передачі даних, аутентифікацію API за допомогою токенів чи ключів доступу, а також дотримання вимог до захисту персональної інформації користувачів.
-	Надання API для веб-клієнта та мобільного додатка, з підтримкою отримання історичних даних і формування звітів.
IoT-клієнт:
-	Постійний збір даних про рівень радіації з визначеною періодичністю, збереження тимчасових даних у пам'яті пристрою для забезпечення їх доступності у разі втрати зв’язку із сервером.
-	Забезпечення передачі даних на сервер через безпечні протоколи (HTTPS).
-	Реалізація повідомлень про перевищення порогових значень радіації у реальному часі.

3.2	 Scope of Subsequent Releases
Веб-клієнт:
-	Відображення радіаційних даних у реальному часі на інтерактивній карті, включаючи зони підвищеного ризику.
-	Аналіз історичних даних із можливістю формування звітів та порівняння різних періодів чи регіонів.
-	Управління користувачами (рівні доступу, моніторинг активності) та IoT-сенсорами (перевірка стану, оновлення, налаштування).
-	Експорт даних у популярні формати (CSV, PDF, Excel).
Мобільний клієнт:
-	Моніторинг рівня радіації в реальному часі з інтерактивною картою та деталями по зонах.
-	Сповіщення про перевищення допустимих норм (push-нотифікації) з рекомендаціями щодо дій.
-	Доступ до архіву радіаційних даних і автоматична адаптація сповіщень за геолокацією користувача.
-	Інтеграція з іншими IoT-пристроями, підтримка персоналізованих налаштувань і офлайн-доступ до даних.

4.3 Operating Enviroment
Система моніторингу радіації доступна як веб-застосунок і мобільний додаток для Android. Серверна частина побудована на Django (Python) із використанням PostgreSQL для зберігання даних. Веб-застосунок реалізовано на HTML, CSS та JavaScript, забезпечуючи інтерактивний інтерфейс для візуалізації даних.
IoT-клієнт на C++ збирає дані сенсорів радіації та передає їх на сервер через захищені протоколи MQTT або HTTPS.
Мобільний додаток розроблено на Java з функціями моніторингу, історії даних і налаштовуваних сповіщень.
